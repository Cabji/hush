20191008 - Bot Stat Tracking for Automatic Priority

Current development version is still 0.4.0 and I have mostly been trying to deal with small bugs for the last few months. I havent spent much time working on Hush but this extended time without updating and releasing has been spent on letting the software run so I can try to improve the faults it has. During this time I have seen that the most annoying thing that is happening is some bots will transfer 99.9% of a file and then the transfer will fail.

I had options in this scenario. I could just write some code in the DCC failure handler that checks how much of a file is transferred and if it is enough (like, 97% or more) I could tell Hush to accept it as a "completed" download. I didn't choose this option because when files don't complete 100% you get annoying problems like not being able to seek through the file on your media player.

I always wanted to implement a "priority" system for deciding which channel or bots we should request from first so I was going to create this, but then I realized that while I had the software doing something I _didn't_ want to happen, I should create a feature that best deals with that problem first and I can then also add the additional "manual priority" feature afterward.

So I am currently creating the code that will house stats on transfers bots do inside the sqlite database and then when a passive search triggers, Hush will automatically be able to decide what priority the bots get. Again, if you also prefer to do a "manual prioritize" like telling Hush "try to download automatically from this channel first" I will add that later on as well since it will simply be an additional layer to the automatic priority system.

20190105 - v0.3.7 Released

This release mostly fixes the faults with the maximum number of automatic downloads system. Previously if you increased the number of maximum auto-downloads it just wouldn't work correctly. In this release this has (hopefully) been fixed up properly. Hush will start multiple auto-downloads up to a maximum number you set in Options -> Transfers and it does this on a search term basis.

So here's how it works:

  1. You enter your passive searches
  2. The passive search timer triggers (or you hit hte new passive search button in Options)
  3. When Hush finds matches for your searches, it will begin auto-downloading (if you've set it to)
  4. If you search for *macgyver*s03e10* and Hush finds 6 files that match this, it will only start 1 download for that search term. It won't start 3 downloads for *macgyver*s03e10* at the same time
  5. If another search term matches, for example *ready*player*one*1080p*, it will also start downloading the first match for that search term for 2 concurrent auto-downloads (and so on, up to your maximum number of auto-downloads)
  6. When a file for a search downloads successfully the search term is removed from the download queue, removing any pending matches for that search with it. Because we already downloaded a file for that search, so we dont want to download duplicate files right?

About

Hush is an addon for KVIrc. It requires at least KVIrc v4.9.2 or higher from v0.2.6 onwards. For Windows binaries of KVIrc v4.9.x please visit Downloading KVIrc Nightly Binaries for Windows and follow the instructions listed there.

Hush creates a search system for XDCC channels along with automation for a number of aspects related to your media content. It removes the need to:

  • publicly search for content
  • use subscription systems that reside on other computers
  • find content titles by offering a degree of automation to continue content acquisition
  • manually move incoming content into media library locations on local storage

It supports watching multiple IRC networks and channels. It should work with any XDCC channel that uses the default bot announcement message layout.

Hush is a hobby project being developed by one person mostly for personal enjoyment, so updates are sparodic - as time and life permits - but it is classed as "under development" by the author. It is written in KVS, KVIrc's internal scripting language.

For queries or bug reports, please join the #hush channel on I2P IRC.

Important Message for v0.2.6 or lower Users Upgrading Hush

If upgrading, make sure you export your hush settings prior to installing the new version! The upgrade routine will remove your settings file and you will have to enter all your settings again. To export your settings go to Hush -> Options -> Export Settings and save to a file. Import from this file after upgrading.

From v0.2.7 onwards there is a dialog asking if you want to preserve your existing config file when uninstalling or upgrading the script.
